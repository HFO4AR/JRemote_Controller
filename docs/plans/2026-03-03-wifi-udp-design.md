# Wi-Fi UDP 控制功能设计

## 1. 概述

为 JRemote Controller 添加 Wi-Fi UDP 控制功能，支持 AP 模式和局域网模式，与现有 BLE 模式并存。

## 2. 连接模式

| 模式 | 说明 |
|------|------|
| BLE | 蓝牙低功耗连接（现有） |
| AP | 手机连接 ESP32 热点，UDP 通信 |
| 局域网 | 手机与 ESP32 连接同一 Wi-Fi 网络 |

## 3. 通信协议

### UDP 配置
- **数据端口**：1034
- **发现端口**：1035
- **协议**：UDP

### 设备发现
- 手机发送 UDP 广播到端口 1035
- ESP32 响应：`JREMOTE:{设备名称}:{IP}:1034`
- 支持发现多台设备

### 控制数据包（9 字节）
| 字节 | 内容 |
|------|------|
| 0 | 帧头（0xAA 正常，0xEE 急停） |
| 1-2 | 左摇杆 X, Y（-127 ~ 127） |
| 3-4 | 右摇杆 X, Y（-127 ~ 127） |
| 5-8 | 按钮状态位掩码 |

### Ping 机制
- 请求：单字节 `0x70` ('p')
- 响应：单字节 `0x50` ('P')

## 4. 架构设计

### 新增文件
```
android_app/.../wifi/
├── WifiService.kt       # Wi-Fi UDP 连接管理
└── UdpDiscovery.kt     # 设备发现

android_app/.../data/
├── ConnectionMode.kt   # 连接模式枚举
└── DiscoveredDevice.kt # 发现设备模型
```

### 核心类职责

| 类 | 职责 |
|------|------|
| WifiService | UDP 连接建立/断开、数据发送、ping |
| UdpDiscovery | UDP 广播发现、设备列表管理 |
| ConnectionMode | BLE/AP/LAN 枚举定义 |
| DiscoveredDevice | 设备名称、IP、端口、信号强度 |

### 数据流
1. 用户选择模式（BLE/AP/LAN）
2. 扫描/发现设备
3. 用户选择设备并连接
4. ControlViewModel 根据模式调用对应 Service
5. 数据发送统一经过 ControlViewModel

## 5. UI 设计

### 连接界面
- 顶部模式切换下拉菜单：蓝牙 / AP / 局域网
- 扫描/发现设备列表
- 点击设备连接/断开

### 状态显示
- Wi-Fi 模式下显示 IP 地址
- 信号强度（局域网模式）

## 6. 错误处理

- 连接失败：显示错误提示
- 连接断开：显示断开提示，不自动重连
- 权限不足：提示用户授权

## 7. 持久化

保存上次连接信息：
- 连接模式（BLE/AP/LAN）
- 设备地址或 IP
- 自动重连时使用

## 8. ESP32 端要求

### 功能
- AP 模式：作为热点
- Station 模式：连接 Wi-Fi
- 监听 1034 接收控制数据
- 监听 1035 响应发现广播

### 数据格式
- 发现响应：`JREMOTE:{设备名称}:{IP}:1034`
- 控制数据：9 字节（与 BLE 相同）
